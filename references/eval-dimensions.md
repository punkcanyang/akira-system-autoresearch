# AKIRA Skill 评估维度

不同类型的 AKIRA skill 有不同的评估标准。本文件定义各类型的维度、权重和评估方法。

## 通用维度（所有 skill 类型共用）

| 维度 | 权重 | 评估方式 | 说明 |
|------|------|----------|------|
| **架构完整度** | 15% | 文件检查 | SKILL.md frontmatter、README、refs/data/evals 目录存在 |
| **文档质量** | 10% | 文本分析 | README 清晰度、安装说明、跨 agent 适配说明 |
| **规范遵从度** | 10% | 规则检查 | 符合 hermes-skill-authoring-standard |
| **可复现性** | 5% | 操作验证 | 其他 agent 能否按说明安装和使用 |

## 人物视角 skill（akira-person-*）

| 维度 | 权重 | 评估方式 | 说明 |
|------|------|----------|------|
| **视角准确性** | 20% | 与原始材料对比 | 是否抓住了该人的核心关注、方法论、批判对象 |
| **语境重建** | 15% | 场景还原检查 | 是否还原了写作/访谈/演讲/被追问的场景，而不是平铺直叙地复述观点 |
| **工艺模拟** | 15% | craft 分析 | 是否体现了专业 craft（比如 Illich 的制度分析方法，不只是他的结论） |
| **声音区分度** | 5% | 对比分析 | 和其他视角的输出是否明显不同 |
| **未来推演** | 5% | 推演测试 | 能否用该视角对未发生的事做出合理判断 |
| **引用完整性** | 5% | refs 检查 | 是否包含关键原始材料（书籍、访谈、演讲） |

### 评估方法

```
eval_score = (视角准确性 * 0.20)
           + (语境重建 * 0.15)
           + (工艺模拟 * 0.15)
           + (声音区分度 * 0.05)
           + (未来推演 * 0.05)
           + (引用完整性 * 0.05)
           + 通用维度 * 0.35

每个维度评分 0-100。
```

### 评估 prompt 模板

**视角准确性测试：**
```
用 {人物} 的视角回答以下问题：{测试问题}
评分标准：是否体现了该人的核心关注和判断方式？
注意：不是看答案是否"正确"，而是看是否"像这个人会说的"。
```

**语境重建测试：**
```
以下是一段 {人物} 的 {写作/访谈/演讲} 摘录：
{原始文本}

请用这个 skill 生成一段回应，要求：还原该人在什么场景下会说这段话。
评分标准：是否有具体的场景设定（不是"作为一个思想家"这种空泛标签）？
```

**工艺模拟测试：**
```
{人物} 的专业 craft 特征是：{craft 特征列表}
请检查 skill 输出是否体现了这些 craft，而不是只复述观点。
```

## 药物 skill（akira-*，非 person 非 system）

| 维度 | 权重 | 评估方式 | 说明 |
|------|------|----------|------|
| **机制精准度** | 20% | 药理对比 | 受体、代谢通路、剂量效应描述是否准确 |
| **坐标轴对齐** | 15% | Core 规则检查 | 是否遵循 System-Core 的共享坐标系 |
| **回收框架完整度** | 15% | 结构检查 | 维度进入、现实回收、后遗残留（5 面向）是否齐全 |
| **证据等级** | 10% | 来源标注 | 信息是否标注了来源等级（peer-reviewed / anecdotal / speculative） |
| **经验还原** | 10% | 主观描述质量 | 是否有第一人称体验描述（不只是药理数据） |
| **危险提示** | 5% | 风险覆盖 | 药物交互、禁忌、过量风险是否标注 |

## 系统 skill（akira-system-*）

| 维度 | 权重 | 评估方式 | 说明 |
|------|------|----------|------|
| **架构覆盖度** | 20% | 设计检查 | 是否覆盖了所有必要的子系统和接口 |
| **协议完整性** | 15% | 流程检查 | 各阶段是否有明确的输入/输出/判定规则 |
| **边界处理** | 10% | 边界测试 | 异常情况、错误状态、中断恢复是否覆盖 |
| **可组合性** | 10% | 集成检查 | 能否与其他系统 skill 协同工作 |

## 元评估维度

评估工具本身的有效性：

| 维度 | 说明 |
|------|------|
| **预测准确性** | eval 分数与人工评审的一致性（皮尔逊相关系数） |
| **维度覆盖** | 评估维度是否遗漏了重要质量面向 |
| **评分校准** | 不同 skill 的分数分布是否合理（不是全部挤在高分段） |
| **改进建议质量** | eval 给出的改进建议是否可操作 |

### 元评估方法

```
1. 人工评审 N 个 skill，给出质量分数（0-100）
2. 跑 eval，得到 eval 分数
3. 计算相关系数
4. 如果相关系数 < 0.7：调整维度权重或增加维度
5. 如果分数分布不合理：重新校准评分标准
```
